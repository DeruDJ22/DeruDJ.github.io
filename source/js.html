Apa Itu JavaScript (JS)?

JavaScript adalah bahasa pemrograman yang digunakan untuk pengembangan website agar website yang kita punya tuh jadi lebih dinamis dan interaktif brok. Pada tahun 1995 JavaScript diperkenalkan oleh Brendan Eich di Netscape Communications. Nah awalnya tuh JS tuh dibuat cuman untuk kepeluan internal dengan web browser perusahaan tersebut, yaitu Netscape Navigator.

Kegunaan CSS

Kalau sebelumnya kita sudah mengenal tentang HTML dan CSS, sekarang tuh kita akan mengenal tentang JS supaya lebih meningkatkan fungsionalitas halaman web kita. Asal kalian tau ya brok, JS bukan hanya untuk web saja, tapi bisa untuk membuat aplikasi, tools, atau bahkan untuk membuat game. Selain untuk membuat web kita jadi lebih dinamis dan interaktif, JS juga bisa digunakan untuk broses logika brok. Jadi bisa dianalogikan dalam tubuh manusia, JS itu bisa dibilang sendi untuk menggerakkan tubuh kita

Cara Menggunakan JS

Cara menggunakan JS ada 3 macam, yaitu dengan cara JS inline, JS Eksternal, dan JS Internal. Gimana sih cara penulisannya brok?

JS Inline

Yang pertama ada JS Inline.

Apa sih JS Inline itu, brok? JS Inline adalah kode JavaScript yang ditulis langsung di dalam tag HTML, contohnya kayak gini: <button onclick="alert('HELLO!!!')">Klik disini</button> Dari kode itu, kita bisa lihat bahwa tag button itu ada sebuah atribut yang namanya onclick, yang artinya kalo button itu ditekan, bakal muncul alert yang bertuliskan "HELLO!!!". Nah, ada ga sih kelebihan dan kekurangan dari JS Inline? Tentu ada dong brok, berikut kelebihan dan kekurangannya nih brok.

Kelebihan
  • Gampang dan cepet buat diterapin ke satu elemen doang.
  • Mantep buat testing atau debugging fungsi tertentu.
  • Gak perlu ribet nyari-nyari file JS lain buat nambahin fungsi.
Kekurangan
  • Ribet banget kalo websitenya gede brok, karena setiap elemen harus ada atribut JS-nya.
  • Gak efisien, soalnya lu mesti nulis kode yang sama berulang-ulang di setiap elemen.
  • HTML jadi kecampur sama JS, berantakan jadinya.

JS Internal

Selanjutnya ada JS Internal.

Jadi, apa sih JS Internal itu, brok? JS Internal adalah kode JavaScript yang ditulis di dalam tag <script> di dalam file HTML. Contohnya kayak gini: 

<script>
        alert('HELLO dari JS Internal!');
        </script>

Nah, dari kode itu, kita bisa lihat bahwa kita bisa nulis kode JS langsung di dalam HTML. Tapi, ada kelebihan dan kekurangan juga nih, brok. Yuk, kita bahas!

Kelebihan
  • Gampang diimplementasi, tinggal taruh di dalam tag <script>.
  • Kode JS dan HTML ada di satu file, jadi gampang buat diatur.
  • Cocok buat proyek kecil, karena gak perlu file terpisah.
Kekurangan
  • Kalo proyeknya gede, bisa bikin HTML jadi berat, karena semua kode JS ada di satu file.
  • Gak efisien, soalnya kalo ada perubahan, harus edit di setiap halaman.
  • Kalo ada banyak kode, bisa bikin HTML jadi berantakan.

JS Eksternal

Selanjutnya ada JS Eksternal.

Apa lagi itu brok, JS Eksternal? JS Eksternal adalah kode JavaScript yang ditulis di file terpisah, dan kita panggil ke dalam HTML supaya HTML kita bisa nyambung ke JS kita. Cara menggunakannya gampang, kita tinggal pakai tag <script> dengan atribut src untuk memanggil file JS-nya. Contohnya: 

<script src="script.js"></script>

Dari kode di atas, "script" adalah tag untuk memanggil file JS-nya, brok, lalu script.js adalah nama filenya. Nah, di dalam file tersebut, kita bisa nulis semua kode JavaScript yang kita butuhkan. Contoh penulisan di dalam file JS-nya gini brok: 

function greet() {
            alert('HELLO dari JS Eksternal!');
        }
        greet();

Nah, kan sudah kita bahas cara menggunakannya. Tapi ada kelebihan dan kekurangan ga sih brok? Tentu saja ada, berikut kelebihan dan kekurangan dari JS Eksternal.

Kelebihan
  • Kode lebih rapi dan terpisah dari HTML, jadi lebih mudah dibaca.
  • Bisa dipakai di banyak halaman, jadi hemat waktu.
  • Memudahkan dalam debugging, karena semua kode JS ada di satu file.
  • Loading lebih cepat setelah file JS di-cache oleh browser.
Kekurangan
  • Butuh waktu loading tambahan, karena harus memuat file eksternal.
  • Kalo file JS-nya hilang atau error, semua fungsi bisa berantakan.
  • Harus pastikan file JS dipanggil dengan benar di HTML, kalo enggak, bisa error.
  • Keterbatasan akses. Kalo ada yang mau edit kode, harus punya akses ke file JS-nya.
